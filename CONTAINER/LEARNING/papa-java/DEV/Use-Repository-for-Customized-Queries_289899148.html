DevOps : Use Repository for Customized Queries

Created by Nicholas Guner on Apr 20, 2020

If you've been following along, you may have noticed that our repository classes are devoid of any functionality. For example:

@RepositoryRestResource
public interface BookRepository extends JpaRepository<Book, Integer>
{
}

Even though the body of this interface is empty, Spring still generates methods that we can call from our controller of service such as findAll and findById. However, what if we wanted to find all books by some other field, such as title or author. We could call findAll from the controller or service and then filter for the information we want. It would certainly be a lot better if we only fetched database rows that matched our criteria.

We can customize our queries with JPQL, which is a variant of SQL designed specifically for JPA.

Adding Custom Functionality to Repository Classes

We will modify our BookRepository class to be able to query books by title and author:

@RepositoryRestResource
public interface BookRepository extends JpaRepository<Book, Integer>
{
    @Query("select b from Book b where b.title = :title")
    Optional<Book> findByTitle(@Param("title") String title);


    @Query("select b from Book b where b.author = :author")
    Optional<Book> findByAuthor(@Param("author") String bookAuthor);
}

Add New Methods to BookService Class

public  ResponseEntity<Resource<Book>>  getByTitle(String title)
{
    Optional<Book> bookOptional = bookRepository.findByTitle(title);
    if(!bookOptional.isPresent())
    {
        throw new NotFoundException("title-" + title);
    }
    Book book = bookOptional.get();
    Resource<Book> resource = new Resource<>(book);
    ControllerLinkBuilder linkTo = linkTo(methodOn(LibraryController.class).getAllBooks());
    resource.add(linkTo.withRel("all-books"));
    return ResponseEntity.ok(resource);
}
public  ResponseEntity<Resource<Book>>  getByAuthor(String author)
{
    Optional<Book> bookOptional = bookRepository.findByAuthor(author);
    if(!bookOptional.isPresent())
    {
        throw new NotFoundException("author-" + author);
    }
    Book book = bookOptional.get();
    Resource<Book> resource = new Resource<>(book);
    ControllerLinkBuilder linkTo = linkTo(methodOn(LibraryController.class).getAllBooks());
    resource.add(linkTo.withRel("all-books"));
    return ResponseEntity.ok(resource);
}

Add New Methods to the Controller

@GetMapping("/books/title/{title}")
public  ResponseEntity<Resource<Book>>  getBookByTitle(@PathVariable("title") String title)
{
    return bookService.getByTitle(title);
}
@GetMapping("/books/author/{author}")
public  ResponseEntity<Resource<Book>>  getBookByAuthor(@PathVariable("author") String author)
{
    return bookService.getByAuthor(author);
}

Modify BookService.getAll() to Add New Links

public @ResponseBody ResponseEntity<?> getAll()
{
    List<Book> books = bookRepository.findAll().stream()
            .map(b -> {b.add(linkTo(methodOn(LibraryController.class).getAllBooks()).slash("/" + b.getBookId()).withRel("id")); return b;})
            .map(b -> {b.add(linkTo(methodOn(LibraryController.class).getBookByTitle(b.getTitle())).withRel("title")); return b;})
            .map(b -> {b.add(linkTo(methodOn(LibraryController.class).getBookByAuthor(b.getAuthor())).withRel("author")); return b;})
            .collect(Collectors.toList());
    Resources<Book> resources = new Resources<>(books);
    resources.add(linkTo(methodOn(LibraryController.class).getAllBooks()).withSelfRel());
    return ResponseEntity.ok(resources);
}
