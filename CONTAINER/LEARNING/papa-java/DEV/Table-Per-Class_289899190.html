DevOps : Table Per Class

Created by Nicholas Guner on Apr 20, 2020

Table per class is a slight variation to the Mapped Superclass approach. This page will only demonstrated variations from code described in Mapped Superclass and Table Per Class.

Changes to Entity Base Class

The Product class is now marked with @Entity annotation as well as InheritanceType.TABLE_PER_CLASS. Also, the @GeneratedValue annotation needs to use AUTO instead of IDENTITY.

@Entity
@Inheritance(strategy = InheritanceType.TABLE_PER_CLASS)
public abstract class Product extends ResourceSupport
{
    @Id
    @GeneratedValue(strategy = GenerationType.AUTO)
    private int productId;
    private String name;


    public int getProductId()
    {
        return productId;
    }

    public void setProductId(int productId)
    {
        this.productId = productId;
    }

    public String getName()
    {
        return name;
    }

    public void setName(String name)
    {
        this.name = name;
    }

     @Override
    public String toString()
    {
        return "Product{" +
                "productId=" + productId +
                ", name='" + name + '\'' +
                '}';
    }
}

Everything else is exactly the same as in Mapped Superclass and Table Per Class.
