DevOps : Bi-Temporal Design

Created by Nicholas Guner, last modified on Apr 18, 2020

Many teams will have a requirement to save bi-temporal data in the same table with "live" records using in_utc and out_utc fields. Thus, live records will have the out_utc field set to infinity and audit records will have the out_utc value set to some date in the past.

The initial approach to handle this problem was to create a composite-key consisting of the auto-incrementing ID field and the out_utc field. While Hibernate does provide support for composite keys, their use it very unnatural, the coding becomes difficult very quickly and we run into other problems when we try to exploit Hibernate facilities for creating relationship tables.
