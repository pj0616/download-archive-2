Use UTC Time Zone

See UTC

Changes were made to in-memory database to use UTC time zone:

INSERT INTO DUTY(ID, DUTY_ID, DUTY_NAME, DUTY_TYPE, IN_UTC, OUT_UTC, created_by, last_modified_by) VALUES(1, '1', 'duty name 1', 'duty type 1', CURRENT_TIMESTAMP, '9999-12-31T00:00:00Z', 'unknown', 'unknown')

Specified UTC timezone on application.properties

...
hibernate.jdbc.time_zone=UTC
...

When the application is launched, the data is adjusted for local timezone:

Fixing UTC Timezone

In application.properties, we need to specify spring.jpa.properties.hibernate.jdbc.time_zone instead of hibernate.jdbc.time_zone property. (But maybe they mean the same thing)

...
spring.jpa.properties.hibernate.jdbc.time_zone=UTC
...

Now, we no longer need to identify time values as UTC in data.sql file:

Checking in-memory database:

Repository classes need to change to avoid comparisons with CURRENT_TIMESTAMP. Make sure to apply same change to all other repository classes.

@RepositoryRestResource
public interface DutyRepository extends JpaRepository<Duty, Integer>
{
    @Query("select d from Duty d where d.ID = :id and out_utc = TO_DATE('9999-12-31', 'YYYY-MM-DD')")
    Optional<Duty> findById(@Param("id") int id);

    @Query("select d from Duty d where d.ID = :id")
    Optional<Duty> findAuditById(@Param("id") int id);

    @Query("select d from Duty d where d.outUtc = TO_DATE('9999-12-31', 'YYYY-MM-DD')")
    List<Duty> findAll();

    @Query("select d from Duty d where out_utc < TO_DATE('9999-12-31', 'YYYY-MM-DD')")
    List<Duty> findAuditAll();

    @Query("select d from Duty d where d.dutyName = :dutyName and out_utc = TO_DATE('9999-12-31', 'YYYY-MM-DD')")
    List<Duty> findByName(@Param("dutyName") String dutyName);

    @Query("select d from Duty d where d.dutyType = :dutyType and out_utc = TO_DATE('9999-12-31', 'YYYY-MM-DD')")
    List<Duty> findByType(@Param("dutyType") String dutyType);
}
