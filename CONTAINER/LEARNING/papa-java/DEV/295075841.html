Mapped Superclass

Mapped SuperClass is not an "inheritance strategy" but rather a way to factor out common functionality of various Entity objects into a common base class. The common base class is decorated with @MappedSuperclass and may not be an Entity. A database table will be created for each Entity class that inherits from the base superclass.


Types of Entity Inheritance

Mapped Superclass and Table Per Class - factor out common functionality to the base class. One table will be created for each derived entity object

Single Table – the entities from different classes with a common ancestor are placed in a single table

Joined Table – each class has its table and querying a subclass entity requires joining the tables

Table Per Class – In contrast to the MappedSuperclass approach, the superclass is now also an entity which gets mapped to a database table and can be used in queries and associations.
