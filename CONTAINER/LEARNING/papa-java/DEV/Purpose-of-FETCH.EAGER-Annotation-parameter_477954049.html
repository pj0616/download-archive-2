DevOps : Purpose of FETCH.EAGER Annotation parameter

Created by Nicholas Guner on Jun 26, 2020

If we were to execute tests in SegregationRuleClearanceControllerIntegrationWithMocksTest, they would all fail with the following error:

Error:

org.hibernate.LazyInitializationException: failed to lazily initialize a collection of role: com.gs.ep.entity.Rule.ruleDutyMappings, could not initialize proxy - no Session

This problem happens at the following juncture inside the Clearance call as we try to navigate from requested privilege to the associated SoD Rule (if one exists)

  1. From the requested privilege, we can discover associated Duty ID(s).

  2. Once we know the Duty ID, we can ask RuleDutyMapRepository for related Rule-Duty associations:

    List<RuleDutyPair> kerberosRuleDutyPairs = ruleDutyMapRepository.findByDutyId(kerberosDuty.getID());


  3. From RuleDutyPair(s), we can navigate to the associated Rule(s)

    List<Rule> kerberosRules = kerberosRuleDutyPairs.stream()
        .map(RuleDutyPair::getRule)
        .filter(r -> r.getState().equals(RuleState.PREVENT.name()))
        .filter(r -> r.getRuleType().equals(RuleTypeSegregation.SEGREGATION.name()))
        .collect(Collectors.toList());


  4. We would hope that once we have Rule(s) obtained in this fashion, we should be able to freely navigate back to the duties associated with each Rule:

    for (Rule rule : intersection)
{
    List<Duty> duties = rule.getRuleDutyMappings().stream().map(RuleDutyPair::getDuty).collect(Collectors.toList());

    ...
}


  5. However, when we try, we'll get the error shown at the top. If we examine the rule variable in the inspector, we'll see a message which states that ruleDutyMappings could not be evaluated:

    rule = {Rule@14768} "Rule{ID=1, auditId='0001', name='Rule-AB', description='null', ruleType='SEGREGATION', state='PREVENT'}"
 ID = 1
 auditId = "0001"
 name = "Rule-AB"
 description = null
 ruleType = "SEGREGATION"
 state = "PREVENT"
 controlId = 2097
 ruleDutyMappings = {PersistentSet@15022} Unable to evaluate the expression Method threw 'org.hibernate.LazyInitializationException' exception.
...


To summarize above interaction, we see that we can obtain Rule from RuleDutyPair via RuleDutyMapRepository. However, once we have a Rule obtained in this fashion, we can NOT navigate to Rule's duties, as shown is step # 4.

Use FETCH.EAGER Annotation Parameter inside the Rule class.

The problem descried above can be easily fixed in the Rule class by adding FETCH.EAGER annotation parameter: Note that default is FETCH.LAZY.

From:


public class Rule extends BaseEntity 
{
    ...
    @OneToMany(mappedBy = "rule")
    @JsonIgnore
    private Set<RuleDutyPair> ruleDutyMappings;


To:


public class Rule extends BaseEntity 
{
    ...
    @OneToMany(mappedBy = "rule", fetch = FetchType.EAGER)
    @JsonIgnore
    private Set<RuleDutyPair> ruleDutyMappings;
