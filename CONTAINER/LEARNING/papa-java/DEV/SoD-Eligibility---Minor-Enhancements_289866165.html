DevOps : SoD Eligibility - Minor Enhancements

Created by Nicholas Guner on Apr 19, 2020

Do not need @ResponseBody for @RestController

ID Field should appear before all HATEOAS Links

Currently, ID field appears last, after all HATEOAS links:

{
  "ruleName": "Refreshments-Rule",
  "ruleType": "Allow",
  "ruleState": "Draft",
  "propertiesPresentation": [
    {
      "propertiesId": 1,
      "keyStore": {
        "Department": "T123",
        "Level": "VP",
        "Location": "Singapore"
      }
    },
    {
      "propertiesId": 2,
      "keyStore": {
        "Department": "T123",
        "Level": "MD",
        "Location": "Singapore"
      }
    }
  ],
  "links": [
    {
      "rel": "self",
      "href": "http://localhost.gs.com:8081/rules/1"
    }
  ],
  "ID": 1
},

We would like for this field to appear first, as in:

{
  "ID": 1,
  "dutyName": "Refreshments",
  "privilegePresentations": [
    {
      "privilegeName": "Free coffee",
      "ID": 1
    },
    {
      "privilegeName": "Free milk",
      "ID": 2
    },
    {
      "privilegeName": "Free sugar",
      "ID": 3
    }
  ],
  "links": [
    {
      "rel": "self",
      "href": "http://localhost.gs.com:8081/duties/1"
    },
    {
      "rel": "rule",
      "href": "http://localhost.gs.com:8081/rules/1",
      "title": "Refreshments-Rule"
    },
    {
      "rel": "privilege",
      "href": "http://localhost.gs.com:8081/privileges/1",
      "title": "Free coffee"
    },
    {
      "rel": "privilege",
      "href": "http://localhost.gs.com:8081/privileges/2",
      "title": "Free milk"
    },
    {
      "rel": "privilege",
      "href": "http://localhost.gs.com:8081/privileges/3",
      "title": "Free sugar"
    }
  ]
},

Use @JsonProperty Annotation

Remove @JsonGetter("ID") and @JsonSetter("ID") and use @JsonProperty instead:

public class DutyPresentation extends ResourceSupport
{
    @JsonProperty
    private int ID;
    ....
}
