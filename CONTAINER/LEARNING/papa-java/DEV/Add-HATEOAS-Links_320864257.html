Created by Nicholas Guner on Apr 29, 2020

Modify all Entity objects to extend from ResourceSupport

@Entity
public class Subscription extends ResourceSupport
...


Start by adding HATEOAS links to Entity controller - SubscriptionController

First, we will add a HATEOAS link to every Subscription. We will modify getSubscriptions() method to look like this:

@GetMapping("/subscriptions")
public ResponseEntity<Resources<Subscription>> getSubscriptions()
{
    List<Subscription> subscriptions = subscriptionRepository.findAll();
    List<Subscription> linkedSubscriptions = subscriptions.stream()
            .map(s -> {s.add(linkTo(methodOn(getClass()).getSubscriptions(s.getID())).withRel("id")); return s;})
            .collect(Collectors.toList());
    Resources<Subscription> resources = new Resources<>(linkedSubscriptions);
    return ResponseEntity.ok(resources);
}

Note that to use linkTo and mothodOn static methods we need to import:

import static org.springframework.hateoas.mvc.ControllerLinkBuilder.linkTo;
import static org.springframework.hateoas.core.DummyInvocationUtils.methodOn;

Now that we have a view of a single subscription object, we need to get back to be able to view all subscriptions. For this, we will add HATEOAS link to getSubsctions(id) method.

We will add HATEOIS to getSubscriptions(id) method:

@GetMapping("/subscriptions/{id}")
public ResponseEntity<Resource<Subscription>> getSubscriptions(@PathVariable("id") int id)
{
    Optional<Subscription> subscriptionOptional = subscriptionRepository.findById(id);
    if(!subscriptionOptional.isPresent())
    {
        throw new NotFoundException("subscription-" + id);
    }
    Subscription subscription = subscriptionOptional.get();
    Resource<Subscription> resource = new Resource<>(subscription);
    resource.add(linkTo(methodOn(getClass()).getSubscriptions()).withRel("all"));
    return ResponseEntity.ok(resource);
}

Add HATEOAS to EmployeeController

This class requries more work since, in addition to adding links to individual Employee objects, we may also want to add links to related objects, such as Project and Subscriptions.

We will implement EmployeeController.getEmployees method with links to individual Employee objects, as well as links to associated projects and subscriptions:

@GetMapping("/employees")
public ResponseEntity<List<Resource<Employee>>> getEmployees()
{
    List<Employee> employees = employeeRepository.findAll();
    List<Resource<Employee>> employeeResources = new ArrayList<>();
    for(Employee employee : employees)
    {
        Resource<Employee> employeeResource = new Resource<>(employee);
        employeeResource.add(linkTo(methodOn(getClass()).getEmployees(employee.getID())).withRel("id"));
        List<Subscription> subscriptions = employeeSubscriptionMapRepository.findByEmployeeId(employee.getID())
                .stream()
                .map(EmployeeSubscriptionPair::getSubscription)
                .collect(Collectors.toList());
        for(Subscription subscription : subscriptions)
        {
            Link subscriptionLink = linkTo(methodOn(SubscriptionController.class)
                    .getSubscriptions(subscription.getID()))
                    .withRel("subscription")
                    .withTitle(subscription.getSubscriptionCode());
            employeeResource.add(subscriptionLink);
        }
        List<Project> projects = projectEmployeeMapRepository.findByEmployeeId(employee.getID())
                .stream()
                .map(ProjectEmployeePair::getProject)
                .collect(Collectors.toList());
        for(Project project : projects)
        {
            Link projectLink = linkTo(methodOn(ProjectController.class)
                    .getProjects(project.getID()))
                    .withRel("project")
                    .withTitle(project.getProjectName());
            employeeResource.add(projectLink);
        }
        employeeResources.add(employeeResource);
    }
    return ResponseEntity.ok(employeeResources);
}

Note:

In addition to providing the name of the link (i.e. .withRel("project") ), we can also provide another piece of relevant information (i.e. .withTitle(project.getProjectName()).

Add HATEOAS to Mapping Controllers

Mapping controllers, such as ProjectEmployeeMapController, will use Maps of Entity objects, for example:

Map<Project, List<Employee>> groupedMap = projectEmployeePairs.stream()
        .collect(Collectors
                .groupingBy(ProjectEmployeePair::getProject,
                        Collectors.mapping(ProjectEmployeePair::getEmployee, Collectors.toList())));


Therefore, all Simple Entity objects need to implement equals and hashCode methods:

@Override
public boolean equals(Object o)
{
    if (this == o) return true;
    if (o == null || getClass() != o.getClass()) return false;
    if (!super.equals(o)) return false;
    Project project = (Project) o;
    return ID == project.ID &&
            Objects.equals(projectName, project.projectName) &&
            Objects.equals(projectDescription, project.projectDescription);
}
@Override
public int hashCode()
{
    return Objects.hash(super.hashCode(), ID, projectName, projectDescription);
}

When returning collection of objects from Mapping controllers, we will be using Resources class:

List<Resource<Project>> projectResources = new ArrayList<>();
...
Resources<Resource<Project>> resources = new Resources<>(projectResources);
