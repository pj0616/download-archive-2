Create Model and Rest Controller

Refer to package names for Service and RestController classes. We will create a very simple Model and RestController classes.

Model

Book.java

public class Book

{

    long id;

    String name;

    String author;

 

    public Book(long id, String name, String author)

    {

        this.id = id;

        this.name = name;

        this.author = author;

    }

 

    public long getId()

    {

        return id;

    }

 

    public String getName()

    {

        return name;

    }

 

    public String getAuthor()

    {

        return author;

    }

 

    @Override

    public String toString()

    {

        return "Book{" +

                "id=" + id +

                ", name='" + name + '\'' +

                ", author='" + author + '\'' +

                '}';

    }

}

RestController

BookController.java

@RestController

public class BookController

{

    @GetMapping("/books")

    public List<Book> getAllBooks()

    {

        return Arrays.asList(new Book(1l, "Mastering Spring 5.2", "Ranga Karanam"));

    }

}

Test REST API

Debugging

By default, Spring logging is set to the INFO level. If you want to see all configuration that is loaded by Spring Boot, you can set the logging level to DEBUG inside application.properties file:

application.properties

logging.level.org.springframework=debug

After experimenting with the DEBUG option, leave it disabled:

application.properties

#logging.level.org.springframework=debug

Instrumentation

Spring Boot instrumentation is handled by the actuator and hal-browser dependencies in pom.xml file

<dependency>

<groupId>org.springframework.boot</groupId>

<artifactId>spring-boot-starter-actuator</artifactId>

</dependency>

<dependency>

<groupId>org.springframework.data</groupId>

<artifactId>spring-data-rest-hal-browser</artifactId>

</dependency>

Enable All Instrumentation Endpoints

Add following entry into application.properties file:

application.properties

management.endpoints.web.exposure.include=*

Examine Instrumentation Options

After restarting your application with Instrumentation endpoints enabled you may try using following commands:

Command:

curl localhost:8080/actuator/health

Response:

{"status":"UP"}

Change Logging Levels Without Restarting

You can change logging level without restarting your Spring Boot Application

  1. Check current logging level


    Command:

    http://localhost:8080/actuator/loggers/root

    Response:

    {"configuredLevel":"INFO","effectiveLevel":"INFO"}


  2. Use Postman to send a POST command

  5. Check logging level and note change to DEBUG

Command:

http://localhost:8080/actuator/loggers/root

Response:

{"configuredLevel":"DEBUG","effectiveLevel":"DEBUG"}

Checking Info Endpoint

Checking information does not produce any results:

Command:

http://localhost:8080/actuator/info

Response:

{}

We can add Info Endpoint information to application.properties file:

application.properties

info.app.name=@project.name@

info.app.description=@project.description@

info.app.version=@project.version@

info.app.encoding=@project.build.sourceEncoding@

info.app.java.version=@java.version@

After restarting the application, following info information would be displayed:

Making Changes to the Running Application

The DevTools Spring Boot dependency allows making changes to the Model and to observe those changes without restarting the application.

<dependency>

<groupId>org.springframework.boot</groupId>

<artifactId>spring-boot-devtools</artifactId>

<scope>runtime</scope>

<optional>true</optional>

</dependency>

Type REST Command into the browser to observe the current state:

URL:

http://localhost:8080/books

Response:

[{"id":1,"name":"Mastering Spring 5.1","author":"Ranga Karanam"}]

Changes to IntelliJ to support Live Reloads

In IntelliJ, choose "Project Settings", "Build, Execution, Deployment", "Compiler" and check "Build project automatically"

In IntelleJ, press CTRL+SHFT+A and select "Registry…", check "compiler.automake.allow.when.app.running"
