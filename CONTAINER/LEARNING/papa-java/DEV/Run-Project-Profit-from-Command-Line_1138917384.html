DevOps : Run Project-Profit from Command Line

Created by Nicholas Guner, last modified on Jan 27, 2021

When spring.io generates a project, it also creates mvnw, mvnw.cmd files and .mnn directory. These files must be copied to the root folder of the project, i.e.: C:\Users\nick\WslFiles\java\project-profit

Execute application from command line with H2 profile without creating a JAR file

From Dir:

C:\Users\nick\WslFiles\java\project-profit

Command:

mvnw spring-boot:run -Dspring-boot.run.profiles=h2

Note: The process runs on port 8081, instead of 8080 as specified in H2 profile:

Tomcat started on port(s): 8443 (http) 8081 (http) with context path ''

Another option is to create a JAR file

Create a JAR file

From root directory, i.e. C:\Users\nick\WslFiles\java\project-profit, execute:

Command:

mvnw package

This command will create project-profit-0.0.1-SNAPSHOT.jar file in C:\Users\nick\WslFiles\java\project-profit\target directory

Once the jar file is created, run the JAR file using one of the following options:

Cmd:

java -jar -Dspring.profiles.active=h2 target\project-profit-0.0.1-SNAPSHOT.jar
java -jar target\project-profit-0.0.1-SNAPSHOT.jar --spring.profiles.active=h2
