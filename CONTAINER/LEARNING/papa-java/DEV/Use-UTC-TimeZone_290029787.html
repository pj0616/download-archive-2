DevOps : Use UTC TimeZone

Created by Nicholas Guner on Apr 19, 2020

See UTC Time Zone for Fix to SoD Bi-Temporality POC project

See also Incorrect value set in OUT_UTC

Fixes To Database SQL Commands

When we initially populated the DB2 database, we used commands that looked like this:

INSERT INTO DUTY(ID, BI_TEMPORAL_ID, DUTY_NAME, IN_UTC, OUT_UTC, created_by, last_modified_by) VALUES(1, '0001', 'Refreshments', CURRENT_TIMESTAMP, '9999-12-31T00:00:00Z', 'unknown', 'unknown')

Note how we populated in_utc and out_utc fields:

Field

Value

in_utc

CURRENT_TIMESTAMP

out_utc

'9999-12-31T00:00:00Z'

If we were to check how data is stored in the database:

Command:

select in_utc, out_utc from Duty where id = 1

Response


IN_UTC                      OUT_UTC                     
-------------------------------------------------
2019-11-25 09:39:42.201432  9999-12-31 00:00:00.0


This is incorrect since the DB2 database knows nothing about UTC TimeStamps and what we stored in in_utc and out_utc fields is the local time.

We can view the difference between local and UTC timestamps with the following command:

Command:

select to_date('99991231', 'YYYYMMDD') - CURRENT_TIMEZONE as UTC, to_date('99991231', 'YYYYMMDD') as LOCAL from sysibm.sysdummy1

Response:


UTC                         LOCAL                       
-------------------------------------------------
9999-12-31 05:00:00.0       9999-12-31 00:00:00.0


Note that as of 11/27/2019, the UTC time should be 5 hours ahead of the EST.

Thus, to insert data into the DB2 database with UTC timestamps, we should use commands that look like this:

INSERT INTO DUTY(ID, BI_TEMPORAL_ID, DUTY_NAME, IN_UTC, OUT_UTC, created_by, last_modified_by) VALUES(5, '0005', 'UTC Duty', CURRENT_TIMESTAMP - CURRENT_TIMEZONE, TO_DATE('9999-12-31', 'YYYY-MM-DD') - CURRENT_TIMEZONE, 'unknown', 'unknown')

Consider following two records, one that was using local (EST) and the other that was using UTC:

ID  DUTY_NAME    IN_UTC                      OUT_UTC                     
------------------------------------------------------------------
6   Local Duty   2019-11-27 13:11:19.156578  9999-12-31 00:00:00.0       
7   UTC Duty     2019-11-27 18:11:24.73081   9999-12-31 05:00:00.0
