Wrapper Types

Wrapper types for primitives are defined for Number, String, and Boolean

A wrapper type is created when we invoke a function on a primitive type

var quote = "some text written today";
var s = quote.substring(4,8);

This code will internally create a wrapper class and is equivalent to the following:

var temp = new String("some text written today");
var s = temp.substring(4,8);
temp = null;

Problems related to Wrapper types

The temporary object is destroyed after usage

Function hoisting

Differences between function declaration and function expression: For function declaration syntax, a function can be called before it is defined. This is called function hoisting, i.e. pushing to the top of the scope.

Variable Arguments

Extra parameters passed to a function will be ignored. However, all passed in parameters are accessible viaarguments.

function evenOrOdd(num)
{
  document.write(arguments[1]);
  document.write(arguments.length);
  if(num % 2 == 0)
    return "even";
  else
    return "odd";
}

document.write(evenOrOdd(10, 20, 30, 40);

To see how many named arguments a function has use Function.length property:

document.write(evenOrOdd.length);

Function that makes use of variable number of arguments:

function product()
{
    var result = 1;
    for(var i = 0; i < arguments.length; i++)
    {
        result *= arguments[i];
    }
    return result;
}

document.write(product(3,4,5))

Prototypes

Every function has a property called prototype.
If we create an object using this function, all objects will share this prototype
Prototype has properties and methods, thus all object can share properties and methods of the prototype
In above example, Flight.display function would be reused by all instances of the Flight class.

function myAsyncFunction() {
  let promise = new Promise((resolve, reject) => {

      console.log("working asynchronously");
      if(error)
