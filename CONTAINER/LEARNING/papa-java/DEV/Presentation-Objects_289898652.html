DevOps : Presentation Objects

Created by Nicholas Guner on Apr 19, 2020

Presentation objects provide us with flexibility on how data should be displayed to the user. Let us consider the output when we want to display Users along with their Eligibility Rules:

Request

Response

http://localhost:8080/users/rules


...
{
  "userName": "Ivan",
  "userLocation": "MD",
  "userDepartment": "T999",
  "links": [
    {
      "rel": "rule",
      "href": "http://localhost:8080/rules/2",
      "title": "Transport-Rule"
    },
    {
      "rel": "rule",
      "href": "http://localhost:8080/rules/4",
      "title": "BigPerks-Rule"
    },
    {
      "rel": "user",
      "href": "http://localhost:8080/users/3",
      "title": "Ivan"
    }
  ]
},
...


HATEOAS links allow us to specify attributes in addition to the link itself. Thus, we can add a "title" to the Rule:

{
"rel": "rule",
"href": "http://localhost:8080/rules/2",
"title": "Transport-Rule"
},

{
"rel": "rule",
"href": "http://localhost:8080/rules/4",
"title": "BigPerks-Rule"
},

Clearly, displaying the "title" attribute is better than not displaying it since we can avoid clicking on the rule link to view specifics of the rule. However, a single attribute may not be enough. For example, in case of Eligibility Rules, it is also very important to know whether the rule is "Allow" or "Disallow".
