DevOps : Audit Controllers were not properly implemented

Created by Nicholas Guner on May 11, 2020

This was revealed with Integration Tests when testing for biTemporalDelete.

Note that SubscriptionAuditController (and others) use following logic:

@GetMapping("/subscriptions/{id}")
public ResponseEntity<List<Resource<Subscription>>> getSubscriptions(@PathVariable("id") int id)
{
    Optional<Subscription> subscriptionOptional = subscriptionRepository.findById(id);
...

However, inside the SubscriptionResository we see:

public interface SubscriptionRepository extends JpaRepository<Subscription, Integer>
{
    @Query("from Subscription where id =:id and outUtc = " + BaseEntity.DATE_INFINITY_STRING)
    Optional<Subscription> findById(@Param("id") int id);
...

Thus, it will not find records whose outUTC is not set to infinity.

Make following change to repository classes:

public interface SubscriptionRepository extends JpaRepository<Subscription, Integer>
{
...
    @Query("from Subscription where id =:id")
    Optional<Subscription> findAllById(@Param("id") int id);
...
}

Make following change to Audit Controller classes:

@GetMapping("/subscriptions/{id}")
public ResponseEntity<List<Resource<Subscription>>> getSubscriptions(@PathVariable("id") int id)
{
    Optional<Subscription> subscriptionOptional = subscriptionRepository.findAllById(id);
    if(!subscriptionOptional.isPresent())
    {
        throw new NotFoundException("subscription-" + id);
    }
    Subscription liveSubscription = subscriptionOptional.get();
    List<Subscription> subscriptions = subscriptionRepository.findByAuditId(liveSubscription.getAuditId());
    List<Resource<Subscription>> subscriptionResources = createSubscriptionResources(subscriptions);
    return ResponseEntity.ok(subscriptionResources);
}
