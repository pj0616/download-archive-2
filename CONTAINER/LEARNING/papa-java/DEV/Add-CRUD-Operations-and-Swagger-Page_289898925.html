We can try using both annotations on our controller class:

Add REST PUT Command

Create new method in the controller

@PutMapping(path="/users/{id}", consumes = "application/json")
public User updateUser(@PathVariable("id") long id, @Valid @RequestBody User user)
{
    Optional<User> currentUserOptional = userRepository.findById(id);
    if(currentUserOptional.isPresent())
    {
        User currentUser = currentUserOptional.get();
        currentUser.setName(user.getName());
        currentUser.setRole(user.getRole());
        currentUser.setLastupdate(user.getLastupdate());
        return userRepository.save(currentUser);
    }
    return userRepository.save(user);
}

Add REST DELETE Command

Create new method in the controller

@DeleteMapping("/users/{id}")
public String deleteUser(@PathVariable("id") long id)
{
    User user = userRepository.getOne(id);
    if(user != null)
    {
        userRepository.delete(user);
        return "deleted";
    }
    return "not found";
}

Add GET-ONE REST Request

Being able to use HATEOAS links to navigate to a specific item is great. However, if we are interested in seeing information for a known ID, we may want to be able to jump to it without having to sift through all items.

Create new method in the controller

@GetMapping("/users/{id}")
public @ResponseBody User getOne(@PathVariable("id") long id)
{
    Optional<User> currentUserOptional = userRepository.findById(id);
    if(currentUserOptional.isPresent())
    {
        HttpServletRequest httpServletRequest = ((ServletRequestAttributes) RequestContextHolder.currentRequestAttributes()).getRequest();
        User user = currentUserOptional.get();
        user.add(linkTo(UserController.class).slash(httpServletRequest.getRequestURI()).withSelfRel());
        return user;
    }
    return currentUserOptional.get();
}
