DevOps : Obtain SSL Certificate Password from Credential Vault

Created by Nicholas Guner on Apr 19, 2020

Can't use dutycepcert Account

System account dutycepcert does not have the "Owner" entitlement and therefore can't be used.

Exception gets thrown from SSlConfig.customize, at the following call:

SystemAccount systemAccount = credentialVault.getSystemAccount(credRef, systemAccountName);

Create new system account for Storing SSL Password

Refer to Create SSL Certificate.

Add SSL Configuration

Add new class:

@Component
@Profile({"db2"})
public class SSLConfig  implements WebServerFactoryCustomizer<ConfigurableServletWebServerFactory>
{
    @Value("${server.ssl.key-store}")
    private  String keyStore;

    @Value("${server.ssl.key-store-type}")
    private  String keyStoreType;

    @Value("${server.ssl.system.account.credref}")
    String credRef;

    @Value("${server.ssl.system.account.name}")
    String systemAccountName;

    @Autowired
    private CredentialVault credentialVault;

    @Override
    public void customize(ConfigurableServletWebServerFactory serverFactory)
    {
        SystemAccount systemAccount = credentialVault.getSystemAccount(credRef, systemAccountName);
        String password = systemAccount.getPassword();

        Ssl ssl = new Ssl();
        ssl.setKeyStore(keyStore);
        ssl.setKeyStoreType(keyStoreType);
        ssl.setKeyPassword(password);
        ssl.setKeyStorePassword(password);
        serverFactory.setSsl(ssl);
    }
}

Changes to application.properties file with addition of SSL properties

server.http.port=8081
server.port=8443
server.ssl.key-store-type=JKS
server.ssl.key-store=classpath:sod-test.jks
server.ssl.system.account.name=sod_build_test
server.ssl.system.account.credref=9264f331180146bdb758b34369a76f95
