DevOps : Java Problems

Created by Nicholas Guner, last modified on Jan 03, 2021

Create a new Java project

Change to the root project directory, i.e.:

Command:

C:\Users\nick\WslFiles\java

Create project named "java-problems" with default package "problems.java"

Command:

mvn archetype:generate -DgroupId=problems.java -DartifactId=java-problems -DinteractiveMode=false

Build project

Commands

cd java-problems
mvn package

Test the package:

Command

java -cp target/java-problems-1.0-SNAPSHOT.jar problems.java.App

Expected Response:

Hello World!

Make Changes to the Project

Make changes to pox.xml to use version 4 for JUnit and to be able to build from command line:

<dependencies>
  <dependency>
    <groupId>junit</groupId>
    <artifactId>junit</artifactId>
    <version>4.9</version>
    <scope>test</scope>
  </dependency>
  <dependency>
    <groupId>org.apache.commons</groupId>
    <artifactId>commons-lang3</artifactId>
    <version>3.0</version>
  </dependency>
</dependencies>
<build>
  <plugins>
    <plugin>
      <groupId>org.apache.maven.plugins</groupId>
      <artifactId>maven-compiler-plugin</artifactId>
      <version>3.5.1</version>
      <configuration>
        <source>1.8</source>
        <target>1.8</target>
      </configuration>
    </plugin>
  </plugins>
</build>

Troubleshoot

It is possible to make a mistake of committing a project to GitLab prior to creating the .gitignore file. If this happens, it will appear that the .gitignore file is being “ignored”. The reason is because these files are already in git. We need to remove files and add them again with .gitignore file in place

Commands:

git rm -rf --cached .
git add .
