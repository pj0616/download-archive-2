DevOps : Read Data from DB2 Database

Up to this point, the Eligibility Rules POC application was interacting entirely with in-memory H2 database. In this section, we demonstrate how to configure this project to read data from DB2 database tables.

Enable Environment Specific application.properties file

Until this point, we were using a single java Run Configuration to run our project:

Name

EligibilityRules-H2

Main Class

com.springboot.eligibility.eligibilityrules.EligibilityRulesApplication

Use classpath of module

eligibility-rules

To connect to the actual database, we will create another application.properties file with the name of application-db2.properties and specify this profile in the new Run Configuration

Name

EligibilityRules-DB2

Main Class

com.springboot.eligibility.eligibilityrules.EligibilityRulesApplication

Program aruments

  • -spring.profiles.active=db2

Use classpath of module

eligibility-rules

Note that application-db2.properties file will inherit settings from application.properties and override them.

Add DB2 Dependency to pom.xml

<dependency>
   <groupId>com.ibm.db2</groupId>
   <artifactId>db2jcc4</artifactId>
   <version>10.5.7.3.69.49</version>
</dependency>

Create DataSourceConfig Class

@Configuration
@Profile({"db2"})
public class DataSourceConfig
{
    @Value("${datasource.driver-class-name}")
    String dataSourceDriverClass;

    @Value("${datasource.url}")
    String dataSourceUrl;

    @Value("${datasource.username}")
    String dataSourceUserName;

    @Value("${datasource.password}")
    String dataSourcePassword;

    @Bean
    public DataSource getDataSource()
    {
        return DataSourceBuilder
                .create()
                .username(dataSourceUserName)
                .password(dataSourcePassword)
                .url(dataSourceUrl)
                .driverClassName(dataSourceDriverClass)
                .build();
    }
}
