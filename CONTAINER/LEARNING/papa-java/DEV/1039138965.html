DevOps : Ch1: Scale From Zero to Millions of Users

Created by Nicholas Guner on Dec 29, 2020

Non-relational (NoSQL) databases

CouchDB, Neo4j, Cassandra, HBASE, Amazon DynamoDB

Four categories of data

  • key-value stores

  • graph stores

  • column stores

  • document stores

NoSQL databases are good for:

  • super-low latency

  • unstructured data

  • serialize-deserialize data (XML, JSON, YAML)

  • store massive amounts of data

Vertical vs Horizontal Scaling

Vertical - add more RAM, CPU, etc.

Horizontal - add more servers

Database Replication with RDBMS

  • Use Master and Salves.

  • All writes are directed to master which replicates data to slaves

  • All reads are directed to slaves

  • If single slave goes down, reads will be redirected to master

  • If Master goes down, slave may be promoted to master

  • In production, promoting slave to master presents problems. Missing data will need to be recovered by running data recovery scripts.

Improve Load/Response Time

  • add a cache layer

  • store static content (JS, CSS, images, videos) to CDN (content delivery network)

Considerations for using cache

  • not ideal for persisting data

  • expiration policies

  • consistency - data modification and cache not in a single transaction

  • eviction policy - LRU, LFU, FIFO

CDN

  • Cost - CDNs run by 3rd party

  • CDN fallback - coping with CDN failure
